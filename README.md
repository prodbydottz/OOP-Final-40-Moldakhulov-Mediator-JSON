# EmirMoldakhulov40NOSON

## Описание проекта

Проект реализует паттерн Mediator для управления взаимодействием между модулями. Каждый модуль независим и не знает о других напрямую - всю коммуникацию между ними берет на себя медиатор.  

Сценарии взаимодействия (какой модуль получает сообщение от какого события) вынесены в отдельный JSON-файл конфигурации, что обеспечивает гибкость и удобство в использовании.

---

## Зачем используется JSON

### 1. Гибкость

- Сценарии можно добавлять, удалять или изменять без изменения кода модулей и медиатора.  
- Администратор может просто редактировать JSON-файл, чтобы изменить правила взаимодействия модулей.

### 2. Разделение кода и конфигурации

- Код модулей остаётся чистым и независимым, он только реализует интерфейс `IModule`.  
- Логика маршрутизации событий не захардкожена в коде, а хранится отдельно, что упрощает поддержку и расширение системы.

### 3. Соответствие техническим требованиям

Технические требования экзамена:

1. Модуль предоставляет интерфейс для регистрации новых модулей ✅  
2. Взаимодействие через события/сообщения ✅  
3. Гибкий механизм настройки сценариев:
   - Возможность задавать новые сценарии без изменения кода ✅  
   - Сценарии хранятся в конфигурации, редактируемой администратором ✅  
4. Добавление новых модулей с минимальными изменениями ✅  

Использование JSON полностью соответствует пункту 3, так как позволяет менять сценарии без перекомпиляции и без трогания кода модулей.

---

"Sender" — модуль, который отправляет событие

"Receivers" — список модулей, которые должны получить событие

Запуск проекта:

Открыть проект в Visual Studio ~> Убедиться, что JSON-файл scenarios.json находится в папке Config ~> Запустить проект → модули будут взаимодействовать согласно конфигурации

Вывод:
Использование JSON-файла позволяет создать гибкую, расширяемую систему, полностью соответствующую требованиям экзамена по проектированию и паттернам Mediator и SOLID.

## Пример содержимого JSON

```json
{
  "Scenarios": [
    { "Sender": "ModuleA", "Receivers": [ "ModuleB" ] },
    { "Sender": "ModuleB", "Receivers": [ "ModuleA" ] }
  ]
}
